# Visual v3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Waves visibly lapping onto the beach (water OVER the terrain edge), unmistakable directional river streaks, and instantly readable relief (step-edge light line + valley-foot AO).

**Architecture:** A second thin full-screen Shader GameObject (`shore-shader.ts`) sits ABOVE the terrain RTs (depth 500_000) and below the player, drawing only the water↔land boundary band: a foam line whose position oscillates across the band with the wind phase, plus a translucent water film on the sand when the wave advances — alpha 0 everywhere else. The main water shader gains an anisotropic river branch (UV compressed ~3× across the flow, faster phase, higher contrast). The bake gains two overlay stamps per level break on the north edge (light line where the north neighbor is lower, dark AO where it is higher).

**Tech Stack:** Phaser 4.2 Shader GameObject (2nd quad), GLSL ES 1.0, existing flow-field texture + `uWind`, RenderTexture stamps.

**Spec:** `docs/superpowers/specs/2026-07-18-visual-v3-design.md`

## Global Constraints

- Branch `feat/visual-v3`. Never commit to `main`. No `Co-Authored-By`. English imperative commits. Commands in `app/` with bun.
- Exact values (spec): shore quad depth **500_000**; foam band from bilinear waterness ∈ (0.35, 0.80); river anisotropy **~3×** across-flow compression, phase **~2×** faster; light line **3px** north edge, `tint 0xffffff, alpha 0.28`; AO `scaleY 6/32`, `tint 0x000000, alpha min(0.30, 0.10 + diff*0.05)`; pixel snap 4px and posterized output in both shaders.
- All new bake stamps go AFTER the top stamp and BEFORE the existing `rt.render()` (PhaserGame.tsx: top stamp ends ~line 248, `rt.render()` at ~line 260).

- North-neighbor queries use `getWorldTile` (global — seam-free across chunks, same as south walls).
- Frozen: gameplay (current/wind/movement), flow-field encoding, chunk planner, the time-boxed flow-refinement queue from PR #39.
- Gates: `cd app && bunx tsc --noEmit && bun test && bun run build` per task; GLSL visual gate is the owner's browser pass.

### Task 1: Shore foam shader + quad

**Files:**

**Interfaces:**

- [ ] **Step 1: Write `shore-shader.ts`**

```ts
// Espuma de costa: um quad ACIMA do terreno que desenha só a faixa de
// fronteira água<->terra — a onda avança e recua sobre a areia no ritmo do
// vento, com lâmina d'água translúcida e espuma pixelada. Alpha 0 fora da
// faixa. GLSL ES 1.0.

export const SHORE_FRAG = `
#ifdef GL_FRAGMENT_PRECISION_HIGH
precision highp float;
#else
precision mediump float;
#endif

uniform float uTime;
uniform vec2  uScroll;
uniform vec2  uResolution;
uniform vec2  uWind;
uniform sampler2D uFlowTex;

const float TILE = 32.0;
const float FIELD = 160.0;
const float SNAP = 4.0;

float hash(vec2 p) {
  p = fract(p * vec2(123.34, 456.21));
  p += dot(p, p + 45.32);
  return fract(p.x * p.y);
}

vec4 fieldTexelAt(vec2 tile) {
  vec2 wrapped = mod(mod(tile, FIELD) + FIELD, FIELD);
  return texture2D(uFlowTex, (wrapped + 0.5) / FIELD);
}

void main(void) {
  vec2 screenPx = vec2(gl_FragCoord.x, uResolution.y - gl_FragCoord.y);
  vec2 worldPx = screenPx + uScroll;
  vec2 snapped = floor(worldPx / SNAP) * SNAP + SNAP * 0.5;

  // waterness bilinear (canal A): 1 = água, 0 = terra.
  vec2 tpos = snapped / TILE - 0.5;
  vec2 base = floor(tpos);
  vec2 f = tpos - base;
  float w00 = fieldTexelAt(base).a;
  float w10 = fieldTexelAt(base + vec2(1.0, 0.0)).a;
  float w01 = fieldTexelAt(base + vec2(0.0, 1.0)).a;
  float w11 = fieldTexelAt(base + vec2(1.0, 1.0)).a;
  float waterness = mix(mix(w00, w10, f.x), mix(w01, w11, f.x), f.y);

  // Fora da faixa de fronteira (spec: 0.35..0.80): nada.
  if (waterness < 0.30 || waterness > 0.80) {
    gl_FragColor = vec4(0.0);
    return;
  }

  // A linha da onda oscila pela faixa no ritmo do vento: fase espacial ao
  // longo da direção do vento + tempo. windMag ~ [0.2, 1.0].
  float windMag = length(uWind);
  vec2 windDir = windMag > 0.001 ? uWind / windMag : vec2(1.0, 0.0);
  float phase = dot(snapped, windDir) * 0.045 + uTime * (0.0009 + windMag * 0.0012);
  float lap = sin(phase) * 0.5 + sin(phase * 0.37 + 1.7) * 0.5; // vai-e-vem irregular
  // Centro da linha de espuma: varre a faixa (avança na areia quando lap > 0).
  float foamCenter = 0.55 - lap * 0.17; // ~[0.38, 0.72]

  vec3 color = vec3(0.0);
  float alpha = 0.0;

  // Lâmina d'água translúcida sobre a areia quando a onda avançou:
  // pixels do lado da terra (waterness < 0.5) já "alcançados" pela linha.
  if (waterness > foamCenter && waterness < 0.55) {
    color = vec3(0.30, 0.50, 0.80);
    alpha = 0.30;
  }

  // Espuma pixelada na linha da onda (posterizada via hash da célula).
  float d = abs(waterness - foamCenter);
  if (d < 0.055) {
    float sparkle = step(0.35, hash(floor(snapped / SNAP) + floor(uTime * 0.002)));
    color = mix(color, vec3(0.92, 0.96, 1.0), 0.9);
    alpha = max(alpha, 0.85 * sparkle);
  }

  gl_FragColor = vec4(color * alpha, alpha); // premultiplicado p/ blend padrão
}
`;
```

### Task 2: Anisotropic river streaks (`water-shader.ts`)

**Files:**
- Modify: `app/src/lib/world/water-shader.ts` (GLSL only)

**Interfaces:** consumes the existing `flow`, `kindB`, `uTime`, `snapped`, `vnoise` internals of `WATER_FRAG`.

- [ ] **Step 1: Edit the GLSL**

Replace the flow-map block (the `ph0…wave` section) with a branch that keeps the current path for sea and uses anisotropic, faster, higher-contrast sampling for rivers:

```glsl
  // Flow-map: duas fases dente-de-serra defasadas 0.5, cross-fade triangular.
  bool isRiver = kindB >= 0.84;
  float speed = isRiver ? 0.0009 : 0.00045; // rio ~2x mais rápido
  float ph0 = fract(uTime * speed);
  float ph1 = fract(ph0 + 0.5);

  float fmag = length(flow);
  vec2 fdir = fmag > 0.001 ? flow / fmag : vec2(1.0, 0.0);
  vec2 fperp = vec2(-fdir.y, fdir.x);

  vec2 p0 = snapped - flow * ph0 * FLOW_REACH;
  vec2 p1 = snapped - flow * ph1 * FLOW_REACH;
  vec2 uv0;
  vec2 uv1;
  if (isRiver) {
    // Anisotropia: comprime ~3x na perpendicular — streaks alongados
    // na direção da corrente.
    uv0 = vec2(dot(p0, fdir), dot(p0, fperp) * 3.0) / TILE * NOISE_FREQ;
    uv1 = vec2(dot(p1, fdir), dot(p1, fperp) * 3.0) / TILE * NOISE_FREQ;
  } else {
    uv0 = p0 / TILE * NOISE_FREQ;
    uv1 = p1 / TILE * NOISE_FREQ;
  }
  float n0 = vnoise(uv0) * 0.7 + vnoise(uv0 * 2.7) * 0.3;
  float n1 = vnoise(uv1) * 0.7 + vnoise(uv1 * 2.7) * 0.3;
  float w0 = 1.0 - abs(ph0 * 2.0 - 1.0);
  float w1 = 1.0 - w0;
  float wave = n0 * w0 + n1 * w1;
```

And in the color section, raise river contrast — replace the single `color` line with:

```glsl
  float toneAmp = isRiver ? 0.46 : 0.30;
  vec3 color = baseColor * (0.80 + tone * toneAmp);
```

(Keep `tone = floor(wave * 4.0) / 4.0` as-is. Note `kindB`/`flow` are declared BEFORE this block in the current file — the wind composition block — so `isRiver` sees them.)
